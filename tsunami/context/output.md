# Output Standards

## Format
- GitHub-flavored Markdown
- Paragraphs for deliverables, not bullet points
- Alternate paragraphs and tables for research/analysis
- Bold for emphasis. Code blocks for commands.
- No emoji unless the user uses them first.

## Citations
- Inline numeric: `Revenue reached $201B [1]`
- References section at end with URLs
- Never fabricate citations

## Deliverables
- <500 words: message text
- 500-2000 words: summary + file
- >2000 words: file only
- Notes and deliverables are separate files

## Voice
Direct. Professional. Specific with data. No hedging.
